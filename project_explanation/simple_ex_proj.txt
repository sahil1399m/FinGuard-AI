## FinGuard-AI: Multi-Agent Anti-Money Laundering System

Imagine a bank processing thousands of transactions every minute. Manually reviewing each transaction for suspicious activity is nearly impossible, so I built an AI-powered Anti-Money Laundering (AML) system that automatically identifies risky transactions, investigates them, and presents the results through a real-time dashboard.

### How the System Works

Since real banking data is confidential, the project starts with a transaction simulator that continuously generates realistic banking transactions. Most transactions are legitimate, while some are intentionally designed to mimic common money laundering patterns.

Before involving AI, every transaction passes through a set of rule-based fraud detection modules.

#### Geo-Velocity Analysis

The system checks whether a user's location changes unrealistically within a short period. For example, if someone performs a transaction in London and then appears to transact from Mumbai a few minutes later, the system recognizes that such travel is physically impossible and flags the activity.

#### Structuring Detection

Money launderers often avoid regulatory reporting thresholds by splitting large amounts into smaller transfers. The system identifies transactions that consistently occur just below reporting limits and marks them as suspicious.

#### Behavioral Analysis

Each user develops a spending profile over time. If someone who typically makes small transactions suddenly initiates a very large transfer, the system detects this deviation and raises the risk level.

### Risk Scoring Engine

Rather than simply classifying transactions as fraudulent or legitimate, the system assigns a risk score between 0 and 100.

Each suspicious pattern contributes to the final score:

* Geo-velocity anomalies increase the score.
* Structuring behavior increases the score.
* Behavioral abnormalities increase the score.

Based on the final score, transactions are categorized into low, medium, or high-risk groups.

### Multi-Agent AI Investigation

For transactions that exceed the risk threshold, the system launches a multi-agent investigation process using LangGraph.

Three specialized AI agents participate:

**Prosecution Agent**

* Examines evidence supporting money laundering.
* Highlights suspicious patterns and risk indicators.

**Defense Agent**

* Looks for legitimate explanations.
* Considers alternative interpretations of the transaction.

**Judge Agent**

* Reviews arguments from both sides.
* Delivers the final verdict along with reasoning.

This approach makes the investigation process more transparent and explainable compared to a single AI model making decisions independently.

### Network Analysis

Money laundering often involves groups of connected accounts rather than isolated transactions.

The system builds a transaction network graph where:

* Nodes represent accounts.
* Edges represent money transfers.

By visualizing transaction relationships, the system can identify suspicious clusters, hidden connections, and potential laundering rings that may not be obvious from individual transactions alone.

### Real-Time Dashboard

A Streamlit dashboard provides a live view of the entire system.

Users can:

* Monitor incoming transactions.
* View risk scores.
* Observe AI agent reasoning.
* Analyze fraud alerts.
* Explore transaction network visualizations.

The dashboard updates continuously, creating a real-time monitoring experience similar to what financial investigators use.

### Key Outcome

The project combines traditional rule-based fraud detection with AI-driven investigation. Instead of only identifying suspicious transactions, it explains why they are suspicious, provides multiple perspectives through AI agents, and visualizes transaction relationships to support financial crime investigations.
